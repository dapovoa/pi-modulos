## **POST** /v1/ocr

### Request Body

application/json

##### bbox\_annotation\_format

[ResponseFormat](https://docs.mistral.ai/api/endpoint/ocr#operation-ocr_v1_ocr_post_request_bbox_annotation_format_responseformat) \|null

Specify the format that the model must output. By default it will use `\{ "type": "text" \}`. Setting to `\{ "type": "json_object" \}` enables JSON mode, which guarantees the message the model generates is in JSON. When using JSON mode you MUST also instruct the model to produce JSON yourself with a system or a user message. Setting to `\{ "type": "json_schema" \}` enables JSON schema mode, which guarantees the message the model generates is in JSON and follows the schema you provide.

##### confidence\_scores\_granularity

"word"\|"page"

Granularity for confidence scores: 'word' (per-word scores) or 'page' (aggregate only). Defaults to None (no confidence scores) to keep response payload small.

##### document

\*[FileChunk](https://docs.mistral.ai/api/endpoint/ocr#operation-ocr_v1_ocr_post_request_document_filechunk) \| [DocumentURLChunk](https://docs.mistral.ai/api/endpoint/ocr#operation-ocr_v1_ocr_post_request_document_documenturlchunk) \| [ImageURLChunk](https://docs.mistral.ai/api/endpoint/ocr#operation-ocr_v1_ocr_post_request_document_imageurlchunk)

Document to run OCR on

##### document\_annotation\_format

[ResponseFormat](https://docs.mistral.ai/api/endpoint/ocr#operation-ocr_v1_ocr_post_request_document_annotation_format_responseformat) \|null

Specify the format that the model must output. By default it will use `\{ "type": "text" \}`. Setting to `\{ "type": "json_object" \}` enables JSON mode, which guarantees the message the model generates is in JSON. When using JSON mode you MUST also instruct the model to produce JSON yourself with a system or a user message. Setting to `\{ "type": "json_schema" \}` enables JSON schema mode, which guarantees the message the model generates is in JSON and follows the schema you provide.

##### document\_annotation\_prompt

string\|null

Optional prompt to guide the model in extracting structured output from the entire document. A document\_annotation\_format must be provided.

##### extract\_footer

boolean

_Default Value:_`false`

##### extract\_header

boolean

_Default Value:_`false`

##### id

string

##### image\_limit

integer\|null

Max images to extract

##### image\_min\_size

integer\|null

Minimum height and width of image to extract

##### include\_image\_base64

boolean\|null

Include image URLs in response

##### model

\*string\|null

##### pages

array<integer>\|null

Specific pages user wants to process in various formats: single number, range, or list of both. Starts from 0

##### table\_format

"markdown"\|"html"

200

Successful Response

##### document\_annotation

string\|null

Formatted response in the request\_format if provided in json str

##### model

\*string

The model used to generate the OCR.

##### pages

\*[array](https://docs.mistral.ai/api/endpoint/ocr#operation-ocr_v1_ocr_post_responses_200_application-json_pages_ocrpageobject) < [OCRPageObject](https://docs.mistral.ai/api/endpoint/ocr#operation-ocr_v1_ocr_post_responses_200_application-json_pages_ocrpageobject) >

List of OCR info for pages.

##### usage\_info

\*[OCRUsageInfo](https://docs.mistral.ai/api/endpoint/ocr#operation-ocr_v1_ocr_post_responses_200_application-json_usage_info_ocrusageinfo)
