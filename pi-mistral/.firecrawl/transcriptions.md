Create Transcription

## **POST** /v1/audio/transcriptions

### Request Body

application/json

##### context\_bias

array<string>

##### diarize

boolean

_Default Value:_`false`

##### file

[File](https://docs.mistral.ai/api/endpoint/audio/transcriptions#operation-audio_api_v1_transcriptions_post_request_file_file)

The File object (not file name) to be uploaded.
To upload a file and specify a custom file name you should format your request as such:

```
file=@path/to/your/file.jsonl;filename=custom_name.jsonl
```

```
file=@path/to/your/file.jsonl;filename=custom_name.jsonl
```

Otherwise, you can just keep the original file name:

```
file=@path/to/your/file.jsonl
```

```
file=@path/to/your/file.jsonl
```

##### file\_id

string\|null

ID of a file uploaded to /v1/files

##### file\_url

string\|null

Url of a file to be transcribed

##### language

string\|null

Language of the audio, e.g. 'en'. Providing the language can boost accuracy.

##### model

\*string

ID of the model to be used.

##### stream

boolean

_Default Value:_`false`

##### temperature

number\|null

##### timestamp\_granularities

array<"segment"\|"word">

Granularities of timestamps to include in the response.

200

Successful Response

##### language

\*string\|null

##### model

\*string

##### segments

[array](https://docs.mistral.ai/api/endpoint/audio/transcriptions#operation-audio_api_v1_transcriptions_post_responses_200_application-json_segments_transcriptionsegmentchunk) < [TranscriptionSegmentChunk](https://docs.mistral.ai/api/endpoint/audio/transcriptions#operation-audio_api_v1_transcriptions_post_responses_200_application-json_segments_transcriptionsegmentchunk) >

##### text

\*string

##### usage

\*[UsageInfo](https://docs.mistral.ai/api/endpoint/audio/transcriptions#operation-audio_api_v1_transcriptions_post_responses_200_application-json_usage_usageinfo)
